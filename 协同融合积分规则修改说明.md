## ✅ 修改详情

### 原积分规则
- 积分规则字段：`points_rule`
- 选项：`standard`（标准）
- 计算逻辑：已落地转介固定核算 1 个积分

### 新积分规则
- 新增字段：`points_calculate`（INTEGER，默认 1）
- 选项：
  - ✅ **核算积分**（值为 1）
  - ❌ **不核算积分**（值为 0）

### 积分计算逻辑

**创建/更新转介记录时：**

```javascript
let finalPoints = 0;
if (business_status === 'completed' && points_calculate !== 0) {
    finalPoints = 1; // 已落地且核算积分，积 1 分
}
// 如果 points_calculate 为 0（不核算积分），则 finalPoints 为 0
```

**规则说明：**
1. 当选择 **"核算积分"** 时：
   - 已落地转介 → 核算 1 个积分
   - 跟进中/无效 → 0 积分

2. 当选择 **"不核算积分"** 时：
   - 无论什么状态 → 都是 0 积分

## 🌐 前端操作

### 新增转介记录
1. 点击"协同融合"标签
2. 点击"➕ 新增转介"按钮
3. 填写基本信息
4. 在"积分规则"下拉框选择：
   - ✅ 核算积分
   - ❌ 不核算积分
5. 保存

### 编辑转介记录
1. 点击列表中某条记录的"✏️"按钮
2. 修改"积分规则"选项
3. 保存修改

## 📝 注意事项

1. **历史数据兼容性**
   - 所有历史数据默认设置为"核算积分"
   - 不影响现有积分统计

2. **积分统计**
   - 仪表盘统计会自动排除"不核算积分"的记录
   - 个人排名只统计实际积分

3. **状态变更**
   - 从"跟进中"改为"已落地"时：
     - 如果 points_calculate=1，自动计算 1 积分
     - 如果 points_calculate=0，保持 0 积分
