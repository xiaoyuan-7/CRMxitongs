# CRM 系统零成本部署指南

## 一、方案概述

**在你的本地电脑直接运行 CRM，配合 Tailscale 实现远程访问，完全免费。**

---

## 二、适用场景

✅ 个人使用或小团队（<5 人）
✅ 预算有限，想先试用
✅ 数据敏感，不想上云
✅ 接受电脑关机时无法访问

❌ 需要 24 小时不间断访问
❌ 多人同时远程访问
❌ 对稳定性要求极高

---

## 三、快速开始（3 步完成）

### 第 1 步：启动 CRM 系统

**Windows:**
```powershell
# 打开 PowerShell 或 CMD
cd C:\Users\你的用户名\.openclaw\workspace\crm-system\backend
npm start
```

**Mac/Linux:**
```bash
cd /Users/你的用户名/.openclaw/workspace/crm-system/backend
# 或
cd /root/.openclaw/workspace/crm-system/backend
npm start
```

**访问地址：** http://localhost:3000

### 第 2 步：安装 Tailscale

**Windows/Mac:**
1. 下载：https://tailscale.com/download
2. 安装并登录（支持 Google/Microsoft/Apple 账号）
3. 记住你的 Tailscale IP（如 100.64.x.x）

**Linux:**
```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
# 访问链接登录
```

### 第 3 步：远程访问

**在手机/其他设备安装 Tailscale:**
1. 下载 Tailscale App
2. 登录**同一账号**
3. 访问：`http://你的TailscaleIP:3000`

## 五、自动化配置

### 5.1 开机自启动（Windows）

**创建批处理文件 `start-crm.bat`:**
```batch
@echo off
cd /d C:\Users\你的用户名\.openclaw\workspace\crm-system\backend
npm start
```

**添加到任务计划程序:**
1. 打开"任务计划程序"
2. 创建基本任务
3. 触发器：开机时
4. 操作：启动程序 → 选择 `start-crm.bat`
5. 勾选"不管用户是否登录都要运行"

### 5.2 开机自启动（Mac）

**创建 LaunchAgent:**
```bash
mkdir -p ~/Library/LaunchAgents
```

**创建文件 `~/Library/LaunchAgents/com.crm.local.plist`:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.crm.local</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/npm</string>
        <string>start</string>
    </array>
    <key>WorkingDirectory</key>
    <string>/Users/你的用户名/.openclaw/workspace/crm-system/backend</string>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
</dict>
</plist>
```

**加载:**
```bash
launchctl load ~/Library/LaunchAgents/com.crm.local.plist
```

### 5.3 开机自启动（Linux）

**创建 systemd 服务:**
```bash
sudo nano /etc/systemd/system/crm.service
```

**内容:**
```ini
[Unit]
Description=CRM System
After=network.target

[Service]
Type=simple
User=你的用户名
WorkingDirectory=/root/.openclaw/workspace/crm-system/backend
ExecStart=/usr/bin/npm start
Restart=always

[Install]
WantedBy=multi-user.target
```

**启用:**
```bash
sudo systemctl enable crm
sudo systemctl start crm
```

## 六、数据备份

### 6.1 手动备份
```bash
# 复制数据库文件
cp /root/.openclaw/workspace/crm-system/backend/data/crm.db /backup/crm_$(date +%Y%m%d).db
```

### 6.2 自动备份（每天凌晨 2 点）

**Windows 任务计划程序:**
- 触发器：每天 02:00
- 操作：`copy "C:\...\crm.db" "D:\backup\crm_%date:~0,4%%date:~5,2%%date:~8,2%.db"`

**Mac/Linux crontab:**
```bash
crontab -e
# 添加：
0 2 * * * cp /root/.openclaw/workspace/crm-system/backend/data/crm.db /backup/crm_$(date +\%Y\%m\%d).db
```

### 6.3 云备份（可选）

**备份到阿里云盘/百度网盘:**
- 使用官方客户端设置自动同步备份文件夹
- 或使用 rclone 命令行工具

## 七、常见问题

### Q1: 电脑关机后还能访问吗？
**A:** 不能。这是零成本方案的限制。如需 24 小时访问，建议购买云服务器或迷你主机。

### Q2: 多人同时访问会卡吗？
**A:** 5 人以内流畅，10 人以上可能变慢。取决于你的电脑配置和带宽。

### Q3: 家里宽带没有固定 IP 怎么办？
**A:** Tailscale 不依赖固定 IP，自动处理 NAT 穿透。

### Q4: 安全吗？别人能访问吗？
**A:** 安全。Tailscale 是私有网络，只有登录同一账号的设备能访问。

### Q5: 数据库会损坏吗？
**A:** 正常关机不会。建议：
- 使用 UPS 电源（约¥100）
- 每天自动备份
- 避免强制断电

### Q6: 如何查看运行日志？
**A:** 
```bash
# Linux/Mac
tail -f /root/.openclaw/workspace/crm-system/backend/logs/app.log

# Windows
Get-Content C:\...\backend\logs\app.log -Tail 50 -Wait
```

## 八、性能优化

### 8.1 数据库优化
```bash
# 每月执行一次 VACUUM
sqlite3 /root/.openclaw/workspace/crm-system/backend/data/crm.db "VACUUM;"
```

### 8.2 清理日志
```bash
# 删除 30 天前的日志
find /root/.openclaw/workspace/crm-system/backend/logs -name "*.log" -mtime +30 -delete
```

### 8.3 限制并发连接
在 `backend/server.js` 中设置：
```javascript
app.set('port', 3000);
// 限制最大并发连接数
server.maxConnections = 50;
```

## 十、总结

**零成本方案适合：**
- ✅ 预算有限
- ✅ 人数少（<5 人）
- ✅ 接受电脑关机时无法访问
- ✅ 想先试用再决定

**不建议长期使用的场景：**
- ❌ 需要 24 小时稳定访问
- ❌ 多人并发使用
- ❌ 数据非常重要不能丢失

**建议：** 先用零成本方案试用 1 个月，确认需求后再考虑升级。
