# 云服务器安全组配置指南

## 问题诊断

✅ 服务器端口监听正常（3000、3001 已开放）
❌ 外网无法访问 → **需要配置云厂商安全组**

---

## 阿里云配置步骤

### 1. 登录阿里云控制台
https://ecs.console.aliyun.com/

### 2. 进入安全组配置
- 左侧菜单：**网络与安全** → **安全组**
- 找到您的实例所在安全组，点击 **配置规则**

### 3. 添加入站规则

点击 **手动添加**，填写以下信息：

| 字段 | 值 |
|------|-----|
| 优先级 | 1 |
| 策略 | 允许 |
| 协议类型 | TCP |
| 端口范围 | 3000/3000 |
| 授权对象 | 0.0.0.0/0 |
| 描述 | CRM 前端访问 |

再点击 **手动添加** 第二条：

| 字段 | 值 |
|------|-----|
| 优先级 | 1 |
| 策略 | 允许 |
| 协议类型 | TCP |
| 端口范围 | 3001/3001 |
| 授权对象 | 0.0.0.0/0 |
| 描述 | CRM 后端 API |

### 4. 保存规则
点击 **保存**，规则立即生效

## 腾讯云配置步骤

### 1. 登录腾讯云控制台
https://console.cloud.tencent.com/cvm/security

### 2. 进入安全组
- 找到您的实例所在安全组
- 点击 **修改规则**

### 3. 添加入站规则

| 字段 | 值 |
|------|-----|
| 类型 | 自定义 |
| 协议端口 | TCP:3000 |
| 来源 | 0.0.0.0/0 |
| 策略 | 允许 |
| 描述 | CRM 前端 |

再添加第二条：

| 字段 | 值 |
|------|-----|
| 类型 | 自定义 |
| 协议端口 | TCP:3001 |
| 来源 | 0.0.0.0/0 |
| 策略 | 允许 |
| 描述 | CRM 后端 API |

### 4. 保存
点击 **保存** 立即生效

## 常见问题

### Q: 配置后还是打不开？
**A:** 等待 1-2 分钟让规则生效，或重启浏览器再试。

### Q: 只能指定 IP 访问吗？
**A:** 可以！将"授权对象"从 `0.0.0.0/0` 改为您的固定 IP（如 `123.45.67.89/32`）。

### Q: 会影响服务器安全吗？
**A:** 只开放必要的端口（3000、3001），且系统有 JWT 认证，风险可控。如需更高安全，可限制授权对象为特定 IP。
