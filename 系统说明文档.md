## 一、系统概述

### 1.1 系统定位

轻量级本地客户管理系统，专为营销团队设计。采用 React + Node.js + SQLite 技术栈，适合轻量级本地部署，无需云端依赖。

### 1.2 核心价值

- ✅ **轻量部署**：SQLite 数据库，无需单独数据库服务器
- ✅ **功能完整**：覆盖客户管理、营销跟进、智能提醒全流程
- ✅ **本地安全**：数据存储在本地，无需担心云端泄露
- ✅ **易于扩展**：模块化设计，便于二次开发

### 1.3 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端 | React | 18 |
| 前端 UI | Ant Design | 5 |
| 图表库 | Recharts | 2.x |
| 路由 | React Router | 6 |
| 后端 | Node.js | 14+ |
| Web 框架 | Express | 4 |
| 数据库 | SQLite | 3 |
| 认证 | JWT + bcrypt | - |

## 二、核心功能模块

### 2.1 营销任务管理

**功能描述：** 营销目标设定与进度追踪

**主要特性：**
- 任务目标设定：可设置有效户、高质量客户的目标数量
- 进度追踪：实时显示完成进度（圆形进度条可视化）
- 储备客户管理：区分已完成和储备中的客户
- 落地预测：本月落地、三个月落地统计

**使用场景：**
- 每月初设定营销目标
- 每日更新客户状态
- 实时查看任务完成进度

---

### 2.2 企业管理

**功能描述：** 企业客户信息全生命周期管理

**主要特性：**
- 企业列表：展示企业名称、关键人、开户状态、代发状态、有效户、高质量、进度状态
- 企业详情：企业简介、财务信息、行业、上下游信息
- 搜索筛选：支持按名称、状态等多维度搜索
- 数据导出：CSV 格式导出

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| name | 文本 | 企业名称 |
| introduction | 文本 | 企业简介 |
| industry | 文本 | 所属行业 |
| financial_info | 文本 | 财务信息 |
| upstream_info | 文本 | 上游供应商信息 |
| downstream_info | 文本 | 下游客户信息 |
| is_account_opened | 布尔 | 是否已开户 |
| is_payroll_service | 布尔 | 是否代发工资 |
| is_active_customer | 布尔 | 是否有效户 |
| is_high_quality | 布尔 | 是否高质量客户 |
| progress_status | 文本 | 营销进度状态 |

**使用场景：**
- 录入新客户信息
- 更新客户状态
- 导出客户数据备份

---

### 2.3 关键人管理

**功能描述：** 企业关键联系人详细信息管理

**主要特性：**
- 多关键人支持：每个企业可添加多个关键人
- 详细信息：出生年月、职位、家庭信息、喜好、推荐送礼
- 主要联系人标记：可设置一个主要联系人
- 生日自动计算：年龄、生肖、下次生日提醒

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| name | 文本 | 姓名 |
| position | 文本 | 职位 |
| birth_date | 日期 | 出生年月 |
| family_info | 文本 | 家庭信息 |
| preferences | 文本 | 个人喜好 |
| gift_recommendations | 文本 | 推荐送礼 |
| is_primary | 布尔 | 是否主要联系人 |

**使用场景：**
- 记录关键人个人信息
- 设置生日提醒
- 送礼建议参考

---

### 2.4 每周计划

**功能描述：** 营销任务周计划安排与执行

**主要特性：**
- 7 天视图：周一至周日任务安排
- 任务分配：可为每天分配企业跟进任务
- 时间段标记：上午/下午/晚上
- 优先级设置：高/中/低优先级
- 联系频次关联：
  - ✅ 本周触达（7 天内有待办完成）
  - ⏰ 两周触达（8-14 天内有待办完成）
  - 📅 本月触达（15-30 天内有待办完成）
  - 🕐 一月前触达（超过 30 天无待办完成）

**自动更新逻辑：**
1. 每周待办完成时 → 自动计算所有企业触达状态
2. 页面首次加载时 → 自动计算一次
3. 根据待办完成日期自动判断触达时间

**使用场景：**
- 每周日制定下周计划
- 每日查看当天任务
- 完成待办后自动更新联系频次

### 4.3 完整 Schema

```sql
-- 企业表
CREATE TABLE companies (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    introduction TEXT,
    industry TEXT,
    financial_info TEXT,
    upstream_info TEXT,
    downstream_info TEXT,
    is_account_opened BOOLEAN DEFAULT FALSE,
    is_payroll_service BOOLEAN DEFAULT FALSE,
    is_active_customer BOOLEAN DEFAULT FALSE,
    is_high_quality BOOLEAN DEFAULT FALSE,
    progress_status TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 关键人表
CREATE TABLE contacts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    company_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    position TEXT,
    birth_date DATE,
    family_info TEXT,
    preferences TEXT,
    gift_recommendations TEXT,
    is_primary BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE
);

-- 营销进度表
CREATE TABLE marketing_progress (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    company_id INTEGER NOT NULL,
    contact_id INTEGER,
    follow_up_date DATE NOT NULL,
    follow_up_type TEXT,
    follow_up_content TEXT,
    next_follow_up_date DATE,
    notes TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE,
    FOREIGN KEY (contact_id) REFERENCES contacts(id) ON DELETE SET NULL
);

-- 提醒表
CREATE TABLE reminders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    contact_id INTEGER,
    company_id INTEGER,
    reminder_type TEXT NOT NULL,
    reminder_date DATE NOT NULL,
    title TEXT NOT NULL,
    description TEXT,
    is_completed BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (contact_id) REFERENCES contacts(id) ON DELETE SET NULL,
    FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE
);

-- 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    role TEXT DEFAULT 'user',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 六、部署指南

### 6.1 环境要求

| 软件 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Node.js | 14.0.0 | 18.x LTS |
| npm | 6.0.0 | 9.x |
| 操作系统 | Windows/macOS/Linux | Linux |

### 6.2 开发环境部署

**步骤 1：安装后端依赖**
```bash
cd /root/.openclaw/workspace/crm-system/backend
npm install
```

**步骤 2：安装前端依赖**
```bash
cd /root/.openclaw/workspace/crm-system/frontend
npm install
```

**步骤 3：启动后端服务**
```bash
cd /root/.openclaw/workspace/crm-system/backend
npm start
```
后端服务将在 http://localhost:3001 启动

**步骤 4：启动前端开发服务器**
```bash
cd /root/.openclaw/workspace/crm-system/frontend
npm start
```
前端将在 http://localhost:3000 启动

### 6.3 生产环境部署

**步骤 1：构建前端**
```bash
cd /root/.openclaw/workspace/crm-system/frontend
npm run build
```

**步骤 2：启动后端（将提供静态文件）**
```bash
cd /root/.openclaw/workspace/crm-system/backend
npm start
```

访问 http://localhost:3001 即可使用系统

### 6.4 环境变量配置

创建 `.env` 文件（backend 目录下）：

```bash
# 服务器端口
PORT=3001

# JWT 密钥（生产环境请修改）
JWT_SECRET=your-production-secret-key

# 数据库路径
DATABASE_PATH=./crm.db

# 日志级别
LOG_LEVEL=info
```

# 清理 npm 缓存
npm cache clean --force
