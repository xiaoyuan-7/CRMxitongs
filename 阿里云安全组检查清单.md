## 详细配置步骤（图文指引）

### 第 1 步：登录阿里云控制台
打开：https://ecs.console.aliyun.com/

### 第 2 步：找到安全组
1. 左侧菜单：**网络与安全** → **安全组**
2. 或者在 **实例列表** 找到您的实例，点击实例 ID
3. 在实例详情页，找到 **安全组** 标签，点击安全组 ID

### 第 3 步：配置入方向规则
1. 点击 **配置规则** 按钮
2. 切换到 **入方向** 标签页（⚠️ 不是出方向！）
3. 点击 **手动添加** 按钮

### 第 4 步：添加规则（逐字对照）

| 配置项 | 必须填写的值 | 常见错误 |
|--------|-------------|---------|
| 优先级 | `1` | 留空或数字太大 |
| 策略 | `允许` | 选成"拒绝" |
| 协议类型 | `TCP` | 选成 UDP 或其他 |
| 端口范围 | `80/80` | 写成 3000 或 8080 |
| 授权对象 | `0.0.0.0/0` | 写成 127.0.0.1 或其他 IP |
| 描述 | `CRM 系统访问` | 可选填 |

### 第 5 步：保存
点击 **保存** 按钮，规则立即生效

## 常见问题排查

### Q1: 配置后还是打不开？
**A:** 
1. 等待 1-2 分钟让规则生效
2. 刷新浏览器或清除缓存
3. 确认是"入方向"不是"出方向"

### Q2: 多个安全组怎么办？
**A:** 实例可能关联多个安全组，需要给每个安全组都添加规则。

### Q3: 能看到登录页面但无法加载数据？
**A:** 还需要开放 3001 端口（后端 API）：
- 端口范围：`3001/3001`
- 其他配置同上

### Q4: 授权对象能限制 IP 吗？
**A:** 可以！将 `0.0.0.0/0` 改为您的固定 IP，如 `123.45.67.89/32`，更安全。
